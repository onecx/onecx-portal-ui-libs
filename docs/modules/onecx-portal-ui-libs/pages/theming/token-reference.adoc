= Theme V2 Token Reference
:idprefix:
:idseparator: -
:toc: macro
:toclevels: 3

This document provides a complete reference of all tokens in the OneCX Theme V2 schema. Each token is listed with its CSS variable name(s) and purpose.

toc::[]

[#naming-convention]
== CSS Variable Naming Convention

All theme tokens are injected as CSS custom properties on `<html>` following this pattern:

[source]
----
--onecx-theme-{path-with-dots-replaced-by-dashes}
----

For **dual-mode colors** (tokens with separate light/dark values), three variables are set:

[cols="2,3"]
|===
| Variable | Value

| `--onecx-theme-{path}-light`
| Light mode value

| `--onecx-theme-{path}-dark`
| Dark mode value

| `--onecx-theme-{path}`
| `light-dark(var(--onecx-theme-{path}-light), var(--onecx-theme-{path}-dark))`
|===

Single-value tokens produce only `--onecx-theme-{path}` with the direct value.

Throughout this document, tokens marked with *(+light/dark)* support dual-mode and produce three CSS variables as shown above.

[#primitives-border-component-variants]
=== Component-Specific Border Variants

Each component variant has the same properties as the default variant above (`color`, `width`, `style`, `radius`, `offset`).

Pattern: `--onecx-theme-primitives-border-variant-{component}-{property}`

[cols="3,4,4"]
|===
| Token Path Pattern | CSS Variable Pattern | Description

| `primitives.border.variant.button.*`
| `--onecx-theme-primitives-border-variant-button-{property}`
| Border overrides specific to buttons

| `primitives.border.variant.input.*`
| `--onecx-theme-primitives-border-variant-input-{property}`
| Border overrides specific to input fields

| `primitives.border.variant.card.*`
| `--onecx-theme-primitives-border-variant-card-{property}`
| Border overrides specific to cards/panels

| `primitives.border.variant.dialog.*`
| `--onecx-theme-primitives-border-variant-dialog-{property}`
| Border overrides specific to dialogs/modals
|===

Where `{property}` is one of: `color` (+light/dark), `width` (or `width.top`/`right`/`bottom`/`left`), `style`, `radius`, `offset`.

[#variant-css-variable-template]
=== CSS Variable Template

Full CSS variable name for any token in this pattern:

[source]
----
--onecx-theme-{parent}-{state}-{severity}-{property}
----

Where:

* `{parent}` = e.g. `primitives-variant-primary`, `primitives-area-surface`, `primitives-defaultVariant`
* `{state}` = `defaultState` | `state-hover` | `state-active` | `state-selected` | `state-focus`
* `{severity}` = `defaultVariant` | `variant-success` | `variant-info` | `variant-warning` | `variant-danger` | `variant-contrast`
* `{property}` = any leaf from the table above

Each region override can contain:

* `primitives.*` вЂ” overrides any primitive token for that region
* `usages.*` вЂ” overrides any usage token for that region

The tokens within each override follow the exact same structure documented in the sections above.
